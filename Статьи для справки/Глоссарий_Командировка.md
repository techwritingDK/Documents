# Глоссарий по разделу «Командировка» системы управления персоналом

| Термин | Определение |
|---|---|
| Авансовый отчёт | Документ, который сотрудник составляет после командировки. Содержит итоговые данные о расходах и подтверждающие документы: чеки, квитанции, билеты. |
| Бронирование | Подтверждение записи об услуге, которое система получает от внешнего сервиса и хранит вместе со статусом и данными поставщика. |
| Динамический бюджет | Лимит расходов, который автоматически рассчитывается в зависимости от условий поездки — её продолжительности или места назначения. |
| Фиксированный бюджет | Лимит расходов с суммой, заданной вручную. Не зависит от параметров поездки. |
| Командировка | Документ в системе, в котором сотрудник указывает параметры деловой поездки: цель, сроки, место назначения и состав участников. |
| Политика поездок | Набор правил, которые задают допустимые лимиты и параметры для расходов на командировки: классы билетов, максимальную стоимость проживания, суточные. |
| Поставщик услуг | Внешний сервис, который предоставляет варианты для бронирования. |
| Расход | Статья затрат в рамках командировки: например, стоимость билета, проживания или такси. К каждому расходу сотрудник прикладывает подтверждающий документ. |
| Нарушение политики поездок | Ситуация, когда сумма расходов выходит за рамки политики поездок. |
| Статус командировки | Текущее состояние документа в системе. Последовательность статусов: «Черновик» → «На согласовании» → «Согласована» → «Завершена». |
| Уведомление | Сообщение, которое система отправляет участнику командировки при наступлении определённого события. Например, уведомление приходит, когда руководитель согласовал поездку. |
| Политика поездок по умолчанию | Базовая политика поездок, которая применяется ко всем сотрудникам, которым не назначена другая политика. |
| Маршрут согласования | Заранее настроенная цепочка согласующих, которая автоматически запускается при нарушении политики поездок. Определяет, кто и в каком порядке должен одобрить отклонение от лимитов. |
