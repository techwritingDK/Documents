# Управление правами сотрудников

Система позволяет администраторам управлять учётными записями сотрудников, организационной структурой и контролировать доступ к корпоративным сервисам.

---

## Роль администратора организации

Администратор — это пользователь с расширенными правами, который управляет настройками организации и аккаунтами сотрудников. Администратор может:

- управлять аккаунтами сотрудников: добавлять, удалять, редактировать, блокировать и восстанавливать аккаунты, а также просматривать карточки сотрудников;
- добавлять или отзывать права администратора у других сотрудников;
- выстраивать организационную структуру: создавать, переименовывать и удалять подразделения / департаменты / отделы, добавлять в них сотрудников и назначать руководителей;
- работать с группами: создавать группы для совместной работы и управлять их составом;
- увеличивать дисковое пространство на Яндекс Диске для отдельных сотрудников;
- управлять финансами: пополнять общий баланс организации с привязанной банковской карты.

---

## Добавление сотрудников в организацию

Для совместной работы необходимо добавить сотрудников. Это можно сделать несколькими способами.

### Способ 1. Создание аккаунтов вручную

Подходит для добавления небольшого числа сотрудников. Администратор создаёт аккаунты на корпоративном домене организации (например, user@company.ru). Сотрудник получает логин и временный пароль для первого входа. Этот метод не требует наличия у сотрудника личного аккаунта.

### Способ 2. Приглашение сотрудников

Подходит для быстрого начала работы, когда у сотрудников уже есть аккаунт в системе. Администратор отправляет приглашение на email сотрудника. Сотрудник принимает приглашение, присоединяясь к организации со своим аккаунтом. После присоединения администратор может при необходимости назначить сотрудника администратором.

### Способ 3. Массовое добавление через CSV-файл

Подходит для одновременного добавления большого количества сотрудников: например, при первичной настройке или интеграции с HR-системой. В интерфейсе системы можно скачать шаблон CSV-файла с нужными полями. Администратор заполняет файл данными сотрудников и загружает его в систему. Система создаёт аккаунты автоматически.

### Способ 4. Подключение единого входа (SSO)

Подходит для крупных компаний с настроенной инфраструктурой управления идентификацией. Позволяет сотрудникам входить в систему, используя учётные данные из корпоративной системы. Для настройки необходимо подключить федерацию удостоверений. Подробные инструкции доступны для:

- Microsoft Active Directory (AD);
- Microsoft Azure Active Directory (Azure AD);
- Keycloak 18 и 19 (инструкция в разработке).

## Управление доступом и удаление сотрудников

### Отзыв прав администратора

Чтобы отозвать права администратора у другого пользователя:

1. Войдите в аккаунт с правами администратора организации.
2. Перейдите в раздел «Сотрудники».
3. Найдите нужного сотрудника в списке.
4. Выберите опцию «Отозвать права администратора».

> **Примечание:** права владельца организации отозвать невозможно.

### Удаление сотрудника из организации

> **Внимание!** Последствия удаления различаются в зависимости от типа аккаунта.
> - Сотрудник с аккаунтом на корпоративном домене: все данные будут безвозвратно удалены.
> - Приглашённый сотрудник с личным аккаунтом потеряет доступ к ресурсам и данным организации, но его личный аккаунт и данные в нём сохранятся.

Процедура удаления:

1. Войдите в аккаунт с правами администратора организации.
2. Перейдите в раздел «Сотрудники».
3. Найдите нужного сотрудника в списке.
4. Выберите опцию «Удалить».

---

## Альтернативные действия перед удалением

В некоторых случаях полное удаление не требуется. Рассмотрите альтернативы.

### 1. Блокировка аккаунта

Сотрудник потеряет доступ ко всем сервисам организации, но его данные будут сохранены. Аккаунт можно разблокировать в любой момент. Блокировка может пригодиться, если сотрудник находится в отпуске, длительной командировке или проходит служебную проверку.

Процедура блокировки:

1. Войдите в аккаунт с правами администратора организации.
2. Перейдите в раздел «Сотрудники».
3. Найдите нужного сотрудника в списке.
4. Выберите опцию «Заблокировать».

### 2. Удаление из подразделения или группы

Если нужно изменить только организационную структуру — удалите сотрудника из конкретного подразделения или группы, не затрагивая его основной аккаунт в организации.

Процедура удаления:

1. Войдите в аккаунт с правами администратора организации.
2. Перейдите в раздел «Подразделения и группы».
3. Найдите нужное подразделение / группу в списке.
4. Найдите нужного сотрудника в списке.
5. Выберите опцию «Удалить из подразделения / группы».
