# Мобильное приложение для заказа такси и доставки. Описание программы

**Обозначение документа:** МПР.ОП-001

**Составил:** Отдел технической документации

## Аннотация

Приложение предназначено для предоставления пользователям услуг заказа такси, курьерской и грузовой доставки, а также доставки еды из ресторанов и продуктов из магазинов посредством мобильного устройства.

В документе приведены:
- общие сведения о программе;
- функциональное назначение программы;
- описание логической структуры и алгоритмов работы пользователя с приложением;
- входные данные;
- выходные данные.

### 1.2 Программное обеспечение, необходимое для функционирования программы

#### 1.2.1 Серверная часть (backend)

1. **Сервис обработки заказов** — приём заявок от пользователей, подбор исполнителей (водителей, курьеров) по близости и рейтингу, управление статусами заказов в реальном времени, очередью и маршрутизации заявок. Реализован как набор микросервисов на языках программирования C++ и Go.

2. **Геопространственный сервис** — расчёт маршрутов и времени в пути, определение зон доступности тарифов и партнёров, непрерывный трекинг координат водителей и курьеров. Использует Яндекс.Карты API и библиотеку маршрутизации на C++.

3. **Сервис тарификации** — динамический расчёт стоимости с учётом расстояния, выбранного тарифа, загруженности спроса и региона пользователя. Микросервис на Go или Python.

4. **Сервис авторизации и управления профилями** — регистрация и аутентификация по номеру телефона, хранение данных пользователей, водителей, курьеров и партнёров. Аутентификация через SMS OTP (одноразовый пароль).

5. **Платёжный сервис** — проведение транзакций, интеграция с платёжными провайдерами, формирование и хранение квитанций. Изолированный микросервис с повышенными требованиями к безопасности (PCI DSS). Взаимодействует с внешними платёжными провайдерами через REST или gRPC. Транзакции фиксируются в отдельной БД PostgreSQL с поддержкой ACID.

6. **Сервис уведомлений** — отправка push-уведомлений через Firebase Cloud Messaging об изменениях статуса заказа.

7. **Каталог партнёров** — хранение и актуализация меню ресторанов и каталогов магазинов, часов работы. Отдельный сервис с собственной базой PostgreSQL.

8. **API-шлюз** — единая точка входа для мобильных клиентов, балансировки нагрузки, аутентификации запросов, ограничения частоты обращений.

9. **Хранилища данных:**
   - S3-совместимое объектное хранилище — фотографии профилей, документы;
   - PostgreSQL — основная реляционная БД для профилей, заказов, транзакций;
   - Redis — кеш, сессии, очереди, геоиндексы активных исполнителей;
   - Kafka — шина событий между микросервисами (асинхронная обработка, гарантия доставки).

10. **Сервис аналитики и мониторинга ClickHouse** — сбор метрик, логирование событий, анализ данных для тарификации и отчётности, контроль качества поездок и доставок.

#### 1.2.2 Клиентская часть (frontend)

##### 1.2.2.1 Для пользователей мобильных устройств с ОС Android

**Язык и фреймворк:**

- Архитектура: Clean Architecture + MVVM;

**Геолокация и карты:**
- FusedLocation Provider Client для позиционирования (GPS + Wi-Fi);
- Yandex Maps Mobile SDK для отображения карт;
- Фоновый трекинг через Foreground Service (обязателен с Android 8.0).

**Сетевой модуль:**
- OkHttp + Retrofit для HTTP/HTTPS;
- TLS 1.2/1.3 + Certificate Pinning;
- WebSocket (OkHttp) для реального времени;
- Kotlin Coroutines + Flow для асинхронности;
- Сериализация: Gson / Moshi / Protobuf.

**Уведомления:**
- Firebase Cloud Messaging (FCM);
- Notification Channels (Android 8.0+);
- Разрешение POST_NOTIFICATIONS (Android 13+).

**Оплата:**
- Google Pay API;
- Валидация карты на клиенте;
- Токенизация на стороне платёжного провайдера.

**Авторизация:**
- SMS OTP + SMS Retriever API (без READ_SMS);
- Токен в Encrypted SharedPreferences / Android Keystore;
- Biometric Prompt API (отпечаток, лицо).

**Локальное хранилище:**
- Room — история заказов, адреса, профиль;
- DataStore — настройки пользователя;
- Glide / Coil — кеш изображений.

**Навигация:**
- Jetpack Navigation Component + NaviGraph;
- Deep links для открытия экранов из SMS / браузера.

**Аналитика и логирование:**
- App Metrica;
- Firebase Crashlytics;
- WorkManager для фоновой отправки логов.

**Сборка и зависимости:**
- Gradle + Kotlin DSL, Version Catalogs;
- Многомодульная структура проекта;
- Минимальная версия: Android 9.0 (API 28).

**Локализация и доступность:**
- strings.xml + Plurals, поддержка RTL;
- TalkBack (contentDescription), touch target 48×48 dp.

## 2. Функциональное назначение

### 2.1 Назначение программы

Приложение реализует следующие классы задач:

- заказ поездки на такси с выбором тарифа;
- курьерская доставка малогабаритных посылок и документов;
- грузовая доставка крупногабаритных предметов с возможностью вызова грузчиков;
- заказ еды из ресторанов-партнёров;
- заказ продуктов из магазинов-партнёров с доставкой.

### 2.2 Сведения о функциональных ограничениях на применение

Доступность отдельных сервисов и опций приложения может различаться в зависимости от региона использования.

## 3. Описание логической структуры

### 3.1 Алгоритм программы

Приложение обрабатывает команды пользователя, которые он даёт через пользовательский интерфейс, а также данные от серверной инфраструктуры сервиса. Обработка происходит с учётом текущего местоположения пользователя, выбранного сервиса и тарифа, состояния сети и настроек приложения.

### 3.2 Методы

Используемые методы основаны на возможностях ОС Android и ОС iOS, на которых работает приложение. Обмен данными между клиентской и серверной частями осуществляется по протоколам HTTPS и WebSocket.

Приложение имеет клиент-серверную архитектуру и состоит из:
- серверной части (backend);
- клиентской части — пользовательского интерфейса и связанных с ним компонентов (frontend).

**Клиентская часть выполняет следующие функции:**
- обеспечивает взаимодействие с пользователем;
- обеспечивает формирование и отправку запросов к серверу;
- обеспечивает отображение данных, полученных от сервера;
- обеспечивает локальное кеширование данных.

**Серверная часть выполняет следующие функции:**
- обеспечивает приём, разбор и обработку запросов от клиентов;
- обеспечивает подбор исполнителей и расчёт маршрутов;
- обеспечивает проведение платёжных операций;
- обеспечивает хранение данных пользователей, заказов и партнёров.

### 3.3 Модули

#### 3.3.1 Модуль заказа такси

Модуль обеспечивает:
- определение текущего местоположения пользователя;
- ввод адреса назначения вручную или указанием точки на карте;
- отображение доступных тарифов и расчётной стоимости для каждого;
- функцию одновременного выбора нескольких тарифов для поиска ближайшего автомобиля;
- подтверждение заказа и отслеживание водителя на карте в реальном времени;
- отображение данных водителя (имя, рейтинг, марка и номер автомобиля) после принятия заказа.

#### 3.3.2 Модуль доставки

Модуль обеспечивает:
- оформление курьерской доставки малогабаритных посылок и документов;
- оформление грузовой доставки крупногабаритных предметов с подбором автомобиля нужного размера и возможностью вызова грузчиков;
- отслеживание местоположения курьера на карте в реальном времени;
- уведомления об изменении статуса доставки.

#### 3.3.3 Модуль заказа еды и продуктов

Модуль обеспечивает:
- отображение каталога ресторанов-партнёров и магазинов;
- формирование заказа из позиций меню с указанием адреса доставки;
- уведомления о статусе приготовления и доставки заказа.

#### 3.3.4 Модуль геолокации и карт

Модуль обеспечивает:
- определение местоположения пользователя;
- отображение карты;
- фоновое обновление геолокации во время активной поездки;
- построение и отображение маршрута на карте;
- запрос разрешения на геолокацию в момент первого обращения к функции и при последующих запусках приложения.

#### 3.3.5 Модуль оплаты

Модуль обеспечивает:
- проведение платёжных операций;
- ввод и валидацию банковской карты на стороне клиента;
- передачу платёжных данных на сервер в зашифрованном виде — локально данные не хранятся;
- формирование и доставку квитанции по завершении заказа.

#### 3.3.6 Модуль уведомлений

Модуль обеспечивает:
- приём push-уведомлений;
- запрос разрешения на уведомления при первом обращении;
- генерацию локальных уведомлений на стороне клиента;
- поддержку Rich Notifications — уведомлений с изображением маршрута или фото водителя.

#### 3.3.7 Лог событий

Приложение фиксирует ключевые действия пользователя и события приложения. Сбор данных осуществляется через App Metrica. Отчёты об ошибках формируются через App Metrica или Firebase Crashlytics. Логи передаются на сервер асинхронно в фоновом режиме через Background App Refresh, с учётом состояния батареи и сети устройства.

#### 3.3.8 Сервис авторизации

Сервис авторизации обеспечивает:
- безопасность входа в приложение через аутентификацию по номеру телефона (SMS OTP);
- хранение сессионного токена в защищённом хранилище устройства;
- поддержку биометрической аутентификации для повторного входа;
- автоматическое считывание SMS-кода через SMS Retriever API;
- для неавторизованных пользователей доступ к функциям приложения невозможен.

### 3.4 Пользовательский интерфейс

Приложение содержит главный экран с выбором сервиса (Такси, Доставка, Еда / Продукты), экраны оформления заказа, карту с отображением исполнителей и маршрута, экраны истории заказов и профиля пользователя.

Навигация между экранами осуществляется через единый NaviGraph (Jetpack Navigation Component). Поддерживаются deep links — открытие конкретного экрана из внешних источников (SMS, браузер).

### 3.5 Связи программы с другими программами

В процессе работы приложение взаимодействует со следующими внешними системами и сервисами:

- **Серверная инфраструктура сервиса (API-шлюз)** — основной канал обмена данными по протоколам HTTPS и WebSocket;
- **Google Play Services** — геолокация, push-уведомления (FCM), платёжный сервис (Google Pay);
- **Apple Platform Services** — геолокация (CoreLocation), push-уведомления (APNs), платёжный сервис (Apple Pay), биометрическая аутентификация (Face ID / Touch ID);
- **Yandex Maps Mobile SDK** — отображение карт и построение маршрутов;
- **App Metrica** — сбор аналитики и пользовательских событий;
- **Firebase Crashlytics** — сбор отчётов об ошибках;
- **Платёжные провайдеры** — проведение транзакций через API платёжных сервисов.

### 3.6 Алгоритмы работы пользователя с приложением

#### 3.6.1 Заказ такси

1. Пользователь запускает приложение и авторизуется по номеру телефона (при первом запуске).
2. Приложение автоматически определяет текущее местоположение пользователя через модуль геолокации и отображает его на карте.
3. Пользователь вводит адрес назначения в текстовое поле либо указывает точку на карте.
4. Приложение рассчитывает маршрут, отображает доступные тарифы и стоимость поездки для каждого из них. При необходимости пользователь может выбрать несколько тарифов одновременно — система найдёт ближайший доступный автомобиль среди выбранных.
5. Пользователь выбирает тариф и подтверждает заказ. Система подбирает водителя и направляет ему заявку.
6. Пользователь оплачивает поездку через модуль оплаты.
7. После принятия заказа водителем пользователь получает уведомление со сведениями о водителе (имя, рейтинг) и автомобиле (марка, номер). Передвижение автомобиля отображается на карте в реальном времени.
8. По завершении поездки пользователь получает квитанцию на указанный при регистрации email.

#### 3.6.2 Заказ курьерской или грузовой доставки

1. Пользователь переходит в раздел доставки, указывает адрес отправки и адрес получения, дату и время отправки.
2. Пользователь выбирает тип доставки: курьерская (для малогабаритных посылок, документов) или грузовая (для крупногабаритных предметов). При выборе грузовой доставки система предлагает автомобили нужного размера и возможность вызова грузчиков.
3. Система рассчитывает стоимость, пользователь подтверждает заказ.
4. Пользователь отслеживает передвижение курьера на карте и получает уведомления об изменении статуса доставки.
5. По завершении доставки пользователь получает квитанцию на указанный при регистрации email.

#### 3.6.3 Заказ еды из ресторанов и продуктов из магазинов

1. Пользователь переходит в раздел еды или продуктов и выбирает ресторан или магазин из доступного перечня.
2. Пользователь формирует заказ, выбирая позиции из каталога, и указывает адрес доставки.
3. Пользователь подтверждает и оплачивает заказ.
4. Приложение уведомляет о статусе приготовления и доставки.
5. По завершении доставки пользователь получает квитанцию на указанный при регистрации email.

## 4. Входные данные

Входными данными являются:

- **Данные авторизации пользователя** (номер телефона) — вводятся при первом запуске;
- **Адрес подачи и адрес назначения** (для заказа такси и доставки) — вводятся пользователем вручную в текстовом поле либо определяются автоматически по геолокации;
- **Данные геолокации** — поступают автоматически от модуля GPS/GNSS устройства;
- **Выбранный тариф или категория услуги** — задаётся пользователем из предложенного перечня;
- **Содержимое заказа** (позиции из меню ресторана или каталога магазина) — задаётся пользователем в соответствующем разделе приложения;
- **Платёжные данные** — вводятся пользователем в защищённой форме.

---

## 5. Выходные данные

Выходными данными являются:

- **Информация о доступных исполнителях** (водителях, курьерах) с указанием расстояния и расчётного времени прибытия;
- **Расчётная стоимость заказа** — отображается в денежных единицах валюты, установленной для региона пользователя;
- **Маршрут поездки или доставки** — отображается на карте в графическом виде;
- **Текущий статус заказа** — отображается в виде текстовых и графических уведомлений;
- **Сведения о водителе или курьере** (имя, рейтинг, марка и номер автомобиля) — после подтверждения заказа;
- **Квитанция (чек)** о совершённой поездке или доставке — формируется по завершении заказа и направляется пользователю.
